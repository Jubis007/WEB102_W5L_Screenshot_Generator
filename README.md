# Cap! 📸

Ever wonder how applications seamlessly pull in live data and generate dynamic images from the web? I recently wrapped up the Week 5 Lab for my Intermediate Web Development (WEB102) course with CodePath at the University of South Florida, stepping into the world of APIs.

For this lab, I built **Cap!**, a React application that integrates directly with the ApiFlash API to capture high quality, fully customizable screenshots of any website on the internet.

Here are a few of the core technical milestones I tackled during this build:

* 🔐 **API Security:** Configured environment variables (`.env`) to securely hide and manage API access keys outside of the source code.
* ⚙️ **Async Data Fetching:** Utilized `async`/`await` and the Fetch API to send complex, multiple parameter queries to a third party server and handle the JSON responses.
* 🖼️ **Dynamic State Management:** Built a robust React state system to manage user form inputs (adjusting dimensions, blocking ads or cookie banners) and map successful API responses into a continuous, real time image gallery.
* 🎨 **UI/UX Overhaul:** Completely bypassed the default styling to engineer a custom, high contrast Dark Mode dashboard utilizing CSS Flexbox for a responsive, modern layout.

It is incredibly satisfying to bridge the gap between a frontend interface and powerful backend services. Feeling much more confident in my ability to read API documentation and securely implement external tools into my own architecture!
